Formats de fichiers de lames numériques

SVS (Aperio) — Format propriétaire TIFF multi-résolutions développé par Aperio (Leica). Pyramide d'images TIFF avec tuiles carrées (généralement 240×240 pixels). Métadonnées stockées dans le header TIFF. Supporte la compression JPEG ou LZW. Utilisé par les scanners Leica (GT450, AT2, etc.).

DICOM (Digital Imaging and Communications in Medicine) — Standard ouvert défini par le NEMA / DICOM Standards Committee. Utilisé principalement en radiologie (CT, IRM, RX) mais supporte aussi la pathologie (DICOM Pathology). Métadonnées extensives (patient, scanner, protocole). Supporte plusieurs types de compression (JPEG, JPEG2000, JPEG-LS, RLE).

MRXS (Mirax) — Format propriétaire développé par 3DHISTECH (Hongrie) pour les scanners Mirax / P1000 / P250. Hiérarchie propriétaire de tuiles (slides et blocs), optimisée pour le streaming progressif. Index propriétaire stocké dans plusieurs fichiers compagnons.

Résumé

Ce projet explore l'efficacité de différents formats de compression d'image appliqués aux lames histopathologiques (Whole Slide Images — WSI).

Dans un premier temps, un benchmark systématique compare trois formats de compression classiques — JPEG, WebP et JPEG2000 — sur le dataset SVS-TCGA-2048. Les métriques PSNR et SSIM sont mesurées pour chaque format à différents niveaux de qualité (QF), permettant d'identifier le meilleur compromis entre fidélité visuelle et réduction de la taille de fichier.

Les résultats montrent que WebP offre le meilleur rapport qualité/taille pour les lames histopathologiques, suivi de JPEG2000 puis JPEG. La réduction de taille peut atteindre 97% tout en maintenant une qualité visuelle diagnostique acceptable (PSNR supérieur à 30 dB).

Dans un second temps, une expérimentation alternative est menée : la conversion des images vers le format vectoriel SVG via l'outil VTracer. L'objectif est d'évaluer si la représentation vectorielle peut offrir une compression efficace pour ce type d'images. Les résultats indiquent que le SVG est en moyenne 10× plus volumineux que le PNG original sur les images histopathologiques, en raison de leur nature riche en textures qui résiste à la vectorisation.

Méthodes de compression

Compression classique : JPEG, WebP, JPEG2000

JPEG est le standard historique de compression d'image, basé sur la DCT (Discrete Cosine Transform) avec quantification des coefficients. WebP, développé par Google, combine compression intra-frame (VP8) et prédiction pour offrir de meilleurs ratios. JPEG2000 utilise la transformée en ondelettes (DWT) pour une compression multi-résolution plus efficace, particulièrement adaptée aux grandes images comme les WSI.

Le benchmark mesure le PSNR et le SSIM pour chaque format à différents niveaux de qualité (QF = 10, 30, 50, 70, 90), sur le dataset SVS-TCGA-2048 (407 images de 2048×2048 pixels extraites de lames TCGA).



Approche vectorielle : SVG avec VTracer


VTracer convertit les images matricielles en SVG vectoriel pur (chemins <path>) en détectant automatiquement les contours et couleurs. L'objectif est d'évaluer si la représentation vectorielle peut compresser efficacement les lames histopathologiques.


Résultat SVG :

Sur les images histopathologiques riches en textures, le SVG est en moyenne 10× plus volumineux que le PNG original. La vectorisation n'est donc pas adaptée à ce type d'imagerie médicale, car la complexité texturale empêche une représentation économe en primitives vectorielles.

Compression Neurale (CINR)

Une approche expérimentale de compression utilise les Représentations Neurales Implicites (CINR - Compression with Implicit Neural Representations). Contrairement aux codecs classiques, cette méthode entraîne un réseau de neurones (Fourier features + SineCNN) à reproduire l'image à partir de ses seuls poids. Le modèle est sur-ajusté (overfitting) sur une image spécifique, et les poids du réseau constituent alors la représentation compressée.

Cette approche, inspirée du papier de Lee et al. (MICCAI 2024), est particulièrement prometteuse pour les images médicales car elle peut apprendre les caractéristiques spécifiques du domaine histopathologique.

Résultats : Le modèle CINR avec ~1M paramètres atteint 33.8 dB sur un patch 256×256 après 5000 itérations (173s sur M4 Pro). Cependant, la comparaison avec JPEG montre que ce dernier reste supérieur en termes de ratio qualité/compression : JPEG Q=30 donne 35.9 dB à 4.1 MB (ratio 24:1), tandis que CINR 1M float16 donne ~29 dB à 2.1 MB (ratio 48:1).

Conclusion : Bien que les INR offrent une représentation continue (zoom sans pyramide), elles ne sont pas encore compétitives face à JPEG pour le stockage de WSI à grande échelle.

Compression Neuronale — Learned Image Compression (5000×5000)

Une troisième approche explore les méthodes de compression neuronale (learned image compression), qui utilisent des autoencodeurs pré-entraînés pour compresser les images. Contrairement aux méthodes classiques (JPEG, WebP) ou aux INR (CINR) qui encodent une image dans des poids de réseau, ces modèles apprennent un codec universel partagé entre toutes les images.

Deux modèles de l'état de l'art sont testés via la bibliothèque CompressAI : bmshj2018_factorized (autoencodeur simple, ~4M paramètres) et mbt2018_mean (avec hyperprior, ~8M paramètres). Les tests utilisent les poids pré-entraînés sur des images naturelles (DIV2K/CLIC), appliqués en inférence sur une région de 5000×5000 pixels extraite d'une lame histopathologique.

Qu'est-ce que la compression neuronale ?

Contrairement aux codecs classiques (JPEG, WebP) qui appliquent des transformations mathématiques fixes (DCT, ondelettes), la compression neuronale utilise des réseaux de neurones profonds entraînés à compresser n'importe quelle image. L'idée clé est un autoencodeur :

  • Un encodeur (réseau convolutif) transforme l'image en un espace latent compressé,
  • Un décodeur (réseau convolutif symétrique) reconstruit l'image à partir de cet espace latent,
  • Un modèle entropique estime la distribution de probabilité des valeurs latentes pour le codage arithmétique.

L'avantage est que chaque composant apprend automatiquement les structures visuelles pertinentes à partir des données, au lieu d'utiliser des transformations prédéfinies. Le compromis qualité/taille est contrôlé par le paramètre λ (lambda) pendant l'entraînement, qui pondère la distortion (MSE) contre le débit (BPP). Plus λ est grand, plus la qualité est élevée et le fichier volumineux.

Les deux modèles testés diffèrent par leur architecture : bmshj2018_factorized utilise un modèle entropique simple (factorisé, sans dépendances spatiales), tandis que mbt2018_mean ajoute un hyperprior — un second autoencodeur qui capture les corrélations spatiales du latent pour réduire encore le débit.

Analyse

mbt2018_mean domine sur toute la gamme :

  • À PSNR ~33 dB, mbt2018_mean (q=4) = 1 132 KB vs JPEG Q=30 = 1 961 KB → 42% plus petit
  • À PSNR ~35 dB, mbt2018_mean (q=5) = 1 559 KB vs JPEG Q=50 = 2 629 KB → 41% plus petit
  • mbt2018_mean q=1 (377 KB) atteint la même qualité que JPEG Q=10 (1 007 KB) → 2.7× plus petit

bmshj2018_factorized vs mbt2018_mean :

  • mbt2018_mean est systématiquement plus efficace que bmshj2018_factorized à qualité égale
  • À q=3 : mbt2018 = 808 KB (32.3 dB) vs bmshj = 924 KB (31.9 dB) → -13% de taille pour +0.4 dB
  • À q=5 : mbt2018 = 1 559 KB (35.8 dB) vs bmshj = 1 803 KB (35.5 dB) → -14% de taille
  • Cet écart s'explique par l'hyperprior de mbt2018_mean qui modélise mieux les dépendances spatiales

Artefacts visuels :

  • JPEG : artefacts de blocs (8×8 DCT) visibles à basse qualité, perte de détails fins
  • bmshj2018_factorized : flou uniforme, pas de blocs mais perte de texture à basse qualité
  • mbt2018_mean : le plus fidèle, préserve mieux les structures cellulaires grâce à l'hyperprior

Learned Image Compression — Scaling 10000×10000

Pour valider la scalabilité des méthodes neuronales, le benchmark est reproduit sur une région 10000×10000 pixels extraite de la même lame SVS. Cette taille correspond à un sous-ensemble de WSI typique pour une analyse histologique complète.

On utilise donc une stratégie de tiling 4096×4096 : l'image est découpée en 9 tuiles, chaque tuile est encodée/décodée indépendamment sur MPS, puis les reconstructions sont réassemblées. Le BPP total est la somme des bits par tuile, divisé par le nombre total de pixels.

Remarque : le tiling dégrade légèrement la qualité (les bords de tuiles n'ont pas accès au contexte des tuiles voisines), mais l'impact reste faible car le champ récepteur des modèles est déjà très large.

Cohérence des résultats à 4× la taille :

  • Le BPP est légèrement plus bas sur 10000² (0.2646 → 0.0909 pour mbt q=3) : les modèles neuronaux profitent davantage des grandes images car les patterns appris se répètent plus souvent.
  • Le PSNR est légèrement plus haut sur 10000² (33.3 dB → 35.9 dB pour mbt q=3) car les statistiques agrégées sont plus stables.
  • Le gain de mbt2018_mean sur JPEG reste de l'ordre de 40-50% à PSNR équivalent.
  • Le tiling 4096 introduit un overhead négligeable sur la qualité grâce au grand champ récepteur des modèles.

Conclusion sur la scalabilité :

La compression neuronale passe à l'échelle sur des images histologiques de grande taille, même sans fine-tuning. Avec tiling 4096, l'inférence complète d'une qualité prend ~20-35 secondes pour 10000×10000 pixels, ce qui reste compatible avec un usage interactif ou batch sur des sous-régions.
